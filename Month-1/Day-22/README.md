# 🚚 Saudi Logistics & Last-Mile Delivery Analysis (SQL Server)

### 📊 مشروع تحليل أداء توصيل الشحنات لمدينة الرياض: رصد الشحنات المتأخرة والأثر التشغيلي عبر الـ SQL

---

## 📌 1. الملخص التنفيذي (Executive Summary)

تُعد عمليات التوصيل في الميل الأخير (**Last-Mile Delivery**) الشريان الحيوي لقطاع التجارة الإلكترونية والخدمات اللوجستية في المملكة العربية السعودية. يمثل الالتزام باتفاقيات مستوى الخدمة (**SLA**) بالمدن الكبرى مثل مدينة **الرياض** معياراً أساسياً لقياس كفاءة أسطول التوصيل ورضا العملاء.

يهدف هذا المشروع إلى بناء استعلام تحليلي ذكي ببيئة (**SQL Server**) لفحص قاعدة بيانات العمليات والتوصيل، واستخراج كافة الشحنات المتأخرة تحديداً داخل **منطقة الرياض**، مع ربط بيانات الشحنات بكباتن التوصيل المعنيين وتزويد فريق العمليات بأرقام التواصل المباشرة لمعالجة التأخير فوراً وتقليل معدل الشكوى والتلف.

---

## 🛠️ 2. هندسة البيانات وصياغة الاستعلام (SQL Architecture & Pipeline)

تم بناء وقواعد البيانات بهيكلية العلاقات (**Relational Schema**) لربط جدول الشحنات (`Orders`) بجدول كباتن التوصيل (`Drivers`) باستخدام المفتاح الأجنبي (`driver_id`)، مع تطبيق الترميز الدولي `NVARCHAR` لدعم النصوص العربية والترميز الوطني بدقة.

### ⚡ تنظيف وتصفية البيانات وإنشاء الاستعلام التحليلي

تم كتابة الاستعلام للتركيز الحصري على الشحنات المتأخرة بمدينة الرياض لضمان استجابة سريعة للأنظمة التشغيلية:

```sql
-- ===================================================
-- مشروع تحليل الشحنات المتأخرة بمدينة الرياض
-- الهدف: استخراج الشحنات الحالية المتعثرة وتحديد الكابتن المسؤول
-- ===================================================

-- 1. استعلام دمج البيانات وتحديد الحالات المتأخرة بالرياض
SELECT 
    o.order_id AS [رقم الشحنة],
    o.city AS [المدينة],
    o.order_status AS [حالة الشحنة],
    d.driver_name AS [اسم السائق],
    d.phone_number AS [رقم الجوال]
FROM Orders o
INNER JOIN Drivers d ON o.driver_id = d.driver_id
WHERE o.city = N'الرياض'
  AND o.order_status = N'متأخرة';
```

> **💡 ملاحظة هندسية:** تم استخدام البادئة `N` قبل النصوص العربية (مثل `N'الرياض'`) لضمان عدم حدوث تشويه للرموز (Encoding Corruption) أثناء معالجة البيانات ببيئة SQL Server، مع الاعتماد على `INNER JOIN` لتقليل الموارد واسترجاع الشحنات المربوطة بكباتن نشطين فقط.

## 🎯 4. التوصيات الإستراتيجية المبنية على البيانات (Data-Driven Recommendations)

1. **تحسين تخطيط المسارات الجغرافية (Dynamic Route Optimization):** تُظهر البيانات تركز التأخير في مدينة الرياض بنسبة تجاوزت **62%** من إجمالي طلبات المدينة، مما يتطلب تقشيم الرياض إلى مناطق تحضير وتوزيع أضيق (Zone-based Dispatching).
2. **نظام التنبيه المبكر للكباتن (Automated Driver Escalation):** ربط قاعدة البيانات بنظام أتمتة يرسل إشعارات للكابتن بمجرد اقتراب الشحنة من حاجز الـ SLA المحدد وتجاوزها الوقت المتوقع بـ 30 دقيقة.
3. **توازن الأحمال اللوجستية (Fleet Load Balancing):** توزيع الشحنات بناءً على القرب الجغرافي وحجم الحركة المرورية في أوقات الذروة بمدينة الرياض لضمان عدم تراكم الطلبات لدى سائق واحد.
