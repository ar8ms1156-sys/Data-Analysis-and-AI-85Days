## 📌 1. الملخص التنفيذي (Executive Summary)

في تطبيقات ومتاجر التجارة الإلكترونية مثل **منصة سلة (Salla)**، يعتبر تقييم كفاءة الأنشطة التجارية وتكرار المبيعات هو المحرك الأساسي للنمو واستدامة الأرباح.

يهدف هذا المشروع إلى بناء خط استعلام ذكي **(SQL Analytical Pipeline)** لفحص قاعدة بيانات المتاجر `stores_cleaned`، وتتبع أداء الأنشطة التجارية المختلفة. يركز المشروع على تحديد وتحليل الأنشطة الأكثر مبيعا، وحساب **متوسط دخل المتجر الواحد (Avg Revenue Per Store)** بدقة لتوجيه خطط التوسع والاستثمار الرقمي.

## 🛠️ 2. هندسة البيانات وصياغة الاستعلام (SQL Architecture & Data Pipeline)

تم بناء الاستعلام داخل بيئة **SQL Server** بالاعتماد على تقنيات متقدمة لضمان سرعة المعالجة وعدم استهلاك موارد السيرفر:

### ⚡ تنظيف البيانات وإنشاء الأعمدة المحسوبة (Calculated Columns)

تم إضافة الأعمدة المالية الحاكمة لحساب المبيعات ديناميكياً بناءً على تقييمات الطلبات ومتوسط أسعار المنتجات:

```sql
-- إضافة الأعمدة المالية المحدثة
ALTER TABLE stores_cleaned 
ADD avg_Price DECIMAL(10,2), Total_Sales DECIMAL(12,2);

-- احتساب إجمالي المبيعات بناءً على حركة الطلبات
UPDATE stores_cleaned 
SET Total_Sales = total_reviews * avg_Price;
```

> 📝 **ملاحظة هندسية:** تم فحص الداتا الخام المرفوعة وتبيّن استقرار هيكليتها بالكامل، وتم تحويلها إلى جدول معرفة محدد داخل قاعدة بيانات `Salla_Store_DB`.

### ⚡ التجميع والتحليل الفئوي (Categorical Aggregations)

تم استخدام دالات التجميع `SUM` و `COUNT` مع إعادة هيكلة الأرقام عبر `FORMAT` واستبعاد الأكسار العشرية غير الضرورية:

```sql
SELECT 
    business_type_ar,
    COUNT(*) AS total_Stores,
    SUM(total_reviews) AS total_orders,
    FORMAT(SUM(Total_Sales), 'N0') + ' SAR' AS total_revenue,
    FORMAT(SUM(Total_Sales) / COUNT(*), 'N0') + ' SAR' AS avg_sales_per_store
FROM stores_cleaned
GROUP BY business_type_ar
ORDER BY SUM(Total_Sales) DESC;
```
