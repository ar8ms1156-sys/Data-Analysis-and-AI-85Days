# 💳 Saudi Payment Gateway Operations & Revenue Analysis (SQL Server)

### 📊 مشروع تحليل أداء بوابة الدفع السعودية: تقييم معدلات القبول والرفض والأثر التشغيلي عبر الـ SQL

---

## 📌 1. الملخص التنفيذي (Executive Summary)

تعتبر بوابات الدفع الإلكتروني العصب المالي المحرك لقطاع التجارة الإلكترونية في المملكة العربية السعودية. أية إخفاقات أو معدلات رفض مرتفعة في عمليات الدفع تؤدي بشكل مباشر إلى فقدان المبيعات وتراجع تجربة المستخدم.

يهدف هذا المشروع إلى بناء خط استعلام تحليلي ذكي (**SQL Analytical Pipeline**) لفحص قاعدة بيانات المعاملات المالية، وتحليل سلوك طرق الدفع المتاحة (**Mada, Apple Pay, STC Pay, Credit Card**)، ومقارنة العمليات المقبولة (**Accepted**) بالمرفوضة (**Rejected**)، وقياس متوسط القيمة المالية لكل معاملة لاستخراج **الأثر التشغيلي** وتوجيه القرار الفني والتجاري.

## 🛠️ 2. هندسة البيانات وصياغة الاستعلام (SQL Architecture & Pipeline)

تم بناء الاستعلامات واختبارها داخل بيئة **SQL Server** مع الاعتماد على أفضل الممارسات للتحكم في استهلاك الموارد وسرعة المعالجة.

### ⚡ تنظيف البيانات وإنشاء الأعمدة المحسوبة (Data Cleaning & Dynamic Calculations)
تم تطبيق دمج ودوال تجميعية ديناميكية للحصول على إجمالي المبيعات، ومتوسط القيم، وحساب المعاملات المقبولة والمرفوضة لكل طريقة دفع:

```sql
-- استعلام تجميع وتحليل أداء طرق الدفع ومعدلات القبول والرفض
SELECT 
    payment_method AS [طريقة الدفع],
    status AS [حالة العملية],
    COUNT(*) AS [إجمالي العمليات],
    FORMAT(AVG(amount_sar), 'N2') + ' SAR' AS [متوسط المبلغ],
    FORMAT(SUM(amount_sar), 'N2') + ' SAR' AS [إجمالي القيمة]
FROM 
    saudi_payment_transactions
GROUP BY 
    payment_method, 
    status
ORDER BY 
    AVG(amount_sar) DESC;
```

> 💡 **ملاحظة هندسية:** تم إجراء اختبارات استقرار وشيكات الجودة على البيانات الخام للتأكد من خلوها من القيم المفقودة (Null Values) وتوحيد صيغ العملات (SAR) واستبعاد الكسور غير الضرورية عبر `FORMAT()`.

## 🎯 4. التوصيات الإستراتيجية المبنية على البيانات (Data-Driven Recommendations)

1. **معالجة رفض بطاقات Mada:** بطاقات مدى تعتبر الأكثر استخداماً في السوق السعودي، ولذلك فإن وجود **110 عملية مرفوضة** بمتوسط عالي يشير لضرورة التواصل مع البنوك المجمعة (Acquiring Banks) للتحقق من أسباب الرفض الفني.
2. **تحسين قواعد الاحتيال لـ Credit Card:** سجلت البطاقات الائتمانية أعلى معدل رفض (**141 عملية**)، مما يستدعي ضبط حساسية فلتر كشف الاحتيال (Fraud Detection) لتجنب رفض المعاملات الشريفة (False Positives).
3. **تطوير ربط STC Pay:** تحقق STC Pay أعلى متوسط مبيعات للعمليات الناجحة (**4,208.74 SAR**)، مما يجعل تحسين وتحديث الربط البرمجي (API) استثماراً عالياً المردود.
